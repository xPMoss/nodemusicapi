Discography API

GET
GET /users
(Returnerar en lista av alla användare)
GET /users?id={id}
(Returnerar en specifik användare med det id som angivits, eller status 204 No Content om ej finns)
GET /members
(Returnerar en lista av alla medlemmar)
GET /members?id={id}
(Returnerar en specifik medlem med det id som angivits, eller status 204 No Content om ej finns)
GET /albums
(Returnerar en lista av alla album)
GET /albums?id={id}
(Returnerar ett specifik album med det id som angivits, eller status 204 No Content om ej finns)
GET /songs
(Returnerar en lista av alla sånger)
GET /songs?id={id}
(Returnerar en specifik sång med det id som angivits, eller status 204 No Content om ej finns)
GET /songs?album_id={album_id}
(Returnerar alla sånger ur ett specifikt album med det album_id som angivits, eller status 204 No Content om ej finns)

POST
POST /users
(Skapar en ny användare. Lösenordet hashas med hashningsalgoritmen 'sha256'. Validerar skickad data och returnerar status 422 unprocessable entity om data ej angivits rätt)
En användare ska se ut så här:
    {
        "lastname": "XXXXX",
        "firstname": "XXXXX",
        "email": "XXXXX",
        "username": "XXXXX",
        "password": "XXXXX"
    }
POST /login
(Loggar in användare, Lösenordet hashas med hashningsalgoritmen 'sha256' och jämförs med det hashade lösenordet i databasen. Returnerar status 401 unauthorized entity om data ej är rätt)
    {
        "username": "XXXXX",
        "password": "XXXXX"
    }

PUT
PUT /users/{id}
(Updaterar en specifik användare med det id som angivits. Validerar skickad data och returnerar status 422 unprocessable entity om data ej angivits rätt, eller status 204 No Content om ej finns)
Ändringen ska se ut så här:
    {
        "lastname": "XXXXX",
        "firstname": "XXXXX",
        "email": "XXXXX",
        "username": "XXXXX",
        "password": "XXXXX"
    }

DELETE
DELETE /users/{id}
(Tar bort en specifik användare med det id som angivits)
